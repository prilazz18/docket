📖 DocketPro: JavaScript Code Glossary

This document explains the common JavaScript programming keywords, logic structures, and functions used throughout the DocketPro application. It is designed to help you quickly explain the code mechanics during your project defense.

1. Variables and Data Storage

const
Short for "constant". It is used to declare a variable whose value cannot be reassigned. It tells the browser, "Store this piece of data in memory, and lock it so nothing else can accidentally change it."
const maxCases = 500;
let
Used to create a variable that can change over time. We use this for search terms, filter states, or counting numbers heavily in our system.
let currentFilter = 'All';
currentFilter = 'Criminal'; // This is allowed!
[] (Arrays) and {} (Objects)
Arrays []: A list of multiple items grouped together (like a list of cases).
Objects {}: A collection of properties related to one specific thing (like a single Case file, storing its caseNo, title, and status).

2. Control Flow & Logic

if / else if / else
Conditional logic. It tells the computer to execute a certain block of code only if a specific condition is mathematically true. We use this to block unauthorized access or filter case categories.
if (password === 'admin123') { openEditModal(); } else { blockAccess(); }
forEach()
A loop mechanic. It tells the program to take an array (list) of items and perform a specific action for each item inside it. We use this heavily to insert 100 cases into the HTML table at once.
allCases.forEach(caseFile => { createTableRow(caseFile); });
filter() & map()
filter: Sifts through an array and creates a new list containing only the items that pass a test (used in the Search Bar).
map: Translates an array into a new format (used to translate case objects into formatted rows for the CSV Export).

3. The Document Object Model (DOM)

document.getElementById('id_name')
The bridge between JavaScript and HTML. It allows JavaScript to reach into the webpage, grab a specific visual element (like a button or input box), and manipulate it.
const myButton = document.getElementById('login-btn');
addEventListener('click', ...)
It "listens" for a user action—such as a click, a keypress, or a scroll movement—and triggers a function when that action happens. This is what handles all button clicks in DocketPro.
myButton.addEventListener('click', () => { alert("Hello"); });
innerHTML
A property that represents the actual HTML code inside an element. We use JavaScript to overwrite the innerHTML of the calendar and the cases table so the dashboard updates without reloading the page.

4. Advanced Mechanisms

async / await
Handles asynchronous code (tasks that take time, like fetching data from the internet or drawing the 0-100% login bar). await tells the code to pause and "wait" for the internet request to finish before moving on to the next line.
await loginToFirebase();
showDashboard(); // Will not run until the login finishes!
import / export
Part of modern JavaScript (ES Modules). Instead of putting 10,000 lines of code in one file, we split the app into smaller, manageable files (Modules). export makes a function available to the rest of the project, and import brings it into the file that needs it.
import { setupAuth } from './auth.js';
() => {} (Arrow Function)
A shorter, more modern way to write a function in JavaScript. It creates a compact block of code that executes whenever it's called.
